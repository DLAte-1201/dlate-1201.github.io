# 개인 포트폴리오 정적 웹사이트

QA/SQA, AI Evaluation, 자동화 테스트 포트폴리오를 소개하기 위한 개인 홈페이지 템플릿입니다. HTML, CSS, JavaScript만 사용했으며 GitHub Pages에서 바로 배포할 수 있습니다.

## 파일 구성

- `index.html`: 홈, About, Projects, Notes/Blog, Contact 섹션
- `styles.css`: 반응형 레이아웃, 다크/라이트 테마, 접근성 스타일
- `script.js`: 모바일 메뉴, 테마 전환, 연도 표시
- `README.md`: 프로젝트 설명과 배포 방법

## 수정할 내용

실제 배포 전 아래 임시 텍스트를 본인 정보로 바꾸세요.

- `Your Name`
- `your.email@example.com`
- `https://github.com/your-id`
- `https://www.linkedin.com/in/your-id`
- Projects 섹션의 GitHub 저장소 링크

투자 관련 글은 특정 종목 추천이 아니라 학습과 리서치 기록 형태로 작성하는 것을 권장합니다.

## GitHub Pages 배포 방법

1. GitHub에서 새 저장소를 만듭니다.
2. 이 폴더의 `index.html`, `styles.css`, `script.js`, `README.md`를 저장소에 업로드합니다.
3. 저장소의 `Settings`로 이동합니다.
4. 왼쪽 메뉴에서 `Pages`를 선택합니다.
5. `Build and deployment`에서 `Source`를 `Deploy from a branch`로 선택합니다.
6. `Branch`를 `main`, 폴더를 `/root`로 선택한 뒤 저장합니다.
7. 잠시 후 GitHub Pages 주소가 표시됩니다.

## 로컬에서 확인하기

별도 빌드 과정이 필요 없습니다. `index.html` 파일을 브라우저로 열면 바로 확인할 수 있습니다.

## 접근성 기본사항

- 본문 바로가기 링크를 포함했습니다.
- 시맨틱 HTML 구조를 사용했습니다.
- 버튼과 메뉴에 접근성 속성을 추가했습니다.
- 키보드 포커스가 보이도록 스타일을 지정했습니다.
- 사용자의 모션 감소 설정을 고려했습니다.
